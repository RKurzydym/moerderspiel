Name des Spiels

Eine Überschrift für das Spiel. Wird auf den Aufträgen und der Übersichtsseite verwendet.
Spiel-ID (optional)

Eine maschinenlesbare ID für das Spiel. Darf nur Ziffern und die Kleinbuchstaben "a" bis "z" (ohne Umlaute!) enthalten. Muss eindeutig sein, soll leicht zu merken sein. Wird als Twitter-Hashtag verwendet. Wird automatisch generiert, wenn nicht angegeben.

Mailadresse des Game Masters (d.h. DU)

Gib eine Mailadresse an, um den Game Master Code auch per Mail zugeschickt zu bekommen.

Beschreibungstext

Ein Beschreibungstext für das Spiel. Wird den Spielern bei Anmeldung etc. angezeigt.

Aufträge pro Spieler

Sollen die Spieler mehr als ein Leben haben, werden entsprechend viele "Kreise" eröffnet. Jeder Spieler ist in jedem Kreis genau einmal vertreten; man kann jeden Kreis als ein weiteres, parallel laufendes Spiel betrachten, in dem genau die gleichen Leute mitspielen. In jedem Kreis hat man ein Leben; erst wenn man in allen Kreisen ermordet wurde, ist man raus aus dem ganzen Spiel. (In einem Multispiel gibt dies die Kreise pro Unterspiel an!)

Optionen

Ein Testspiel ist dafür da, dass ein unerfahrener Game Master oder Developer ein Spiel mal durchtesten kann. Hier werden Spieler und Morde automatisch eingefügt.

Ein Multispiel sind mehrere normale Spiele mit disjunkten Mitspielergruppen plus ein übergreifendes Spiel, in dem alle vertreten sind.
